# DSCA Data Model

> Every piece of information handled by DSCA must be represented by a common conceptual model.

The purpose of this document is to define the fundamental objects manipulated by the reasoning engine.

---

# Fundamental Principle

DSCA does not reason on files.

DSCA does not reason on diagnostic tools.

DSCA reasons on observations of dynamic systems.

Everything else is merely a source of information.

---

# 1 — Signal

A Signal is the smallest measurable unit known by DSCA.

A Signal represents the evolution of a physical, logical or calculated quantity over time.

Examples:

* Engine speed
* Fuel pressure
* Lambda
* Accelerator pedal position
* Air temperature
* DTC activation
* Torque request

A Signal contains:

* Identifier
* Name
* Unit
* Timestamp series
* Value series
* Acquisition frequency
* Data quality
* Source

A Signal has no interpretation by itself.

---

# 2 — Observation

An Observation represents the complete state of a system at a given instant.

It groups together every available Signal measured at the same timestamp.

An Observation answers the question:

> "What was the system doing at this exact moment?"

---

# 3 — Context

A Context describes the environment in which Observations occur.

Examples:

* Engine temperature
* Vehicle configuration
* Fuel type
* Air conditioning state
* Ambient conditions
* Calibration version

The same Observation may have a completely different meaning depending on its Context.

---

# 4 — Event

An Event represents a significant evolution of one or more Signals.

Examples:

* Pressure drop
* Temperature increase
* Torque limitation
* Knock detected
* Sensor incoherence

Events are detected.

They are never imported.

---

# 5 — System State

A System State represents the combination of:

* Observations
* Context
* Active Events

It describes the complete condition of the system during a specific time interval.

---

# 6 — Behavioral Pattern

A Behavioral Pattern represents a recurring organization of Events.

It is independent from any manufacturer.

It represents how a system behaves.

Examples:

* Fuel starvation under high load
* Thermal limitation
* Ignition instability
* Oscillating pressure regulation

Behavioral Patterns are the foundation of reasoning.

---

# 7 — Causal Hypothesis

A Causal Hypothesis is an explainable interpretation of one or more Behavioral Patterns.

A hypothesis always contains:

* Supporting observations
* Supporting events
* Confidence level
* Missing information
* Possible alternative hypotheses
* Suggested validation tests

A hypothesis is never considered absolute truth.

---

# 8 — Explanation

An Explanation is the human-readable interpretation generated by DSCA.

Its purpose is to explain:

* what happened;
* why the hypothesis was proposed;
* which observations support it;
* what additional tests should be performed.

The objective is transparency.

Never certainty.

---

---

# Fundamental Rules

Rule 1

Everything entering DSCA becomes one or more Signals.

Rule 2

Signals never carry meaning alone.

Rule 3

Meaning emerges from Context.

Rule 4

Events are detected, never stored as truth.

Rule 5

Behavioral Patterns emerge from Events.

Rule 6

Hypotheses emerge from Behavioral Patterns.

Rule 7

Every hypothesis must be explainable.

Rule 8

Every explanation must remain falsifiable.

Rule 9

Human interpretation always has the final authority.
